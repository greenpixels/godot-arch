# Command Line Arguments

GodotArch provides several command line arguments to customize its behavior. This page documents all available options.

## Synopsis

```bash
godot-arch [OPTIONS]
```

## Options

### `-p, --project <PATH>`

**Description:** Path to the Godot project root directory that should be linted.

**Default:** `./` (current directory)

**Example:**

```bash
godot-arch --project /path/to/my/godot/project
godot-arch -p ../my-game
```

**Notes:**

- The path can be absolute or relative
- GodotArch will recursively scan all files in this directory
- The directory must exist, or the tool will panic with an error

---

### `-c, --config <PATH>`

**Description:** Path to the configuration file that defines linting rules and behavior.

**Default:** `./godot-arch.config.yaml`

**Example:**

```bash
godot-arch --config custom-config.yaml
godot-arch -c ./configs/strict.yaml
```

**Notes:**

- The configuration file must be in YAML format
- See the [Configuration Reference](./configuration.md) for details on available options
- If the file doesn't exist or is invalid, the tool will exit with an error

---

### `--report <PATH>`

**Description:** Location where a JSON report should be saved after linting completes.

**Default:** None (no report is generated by default)

**Example:**

```bash
godot-arch --report ./reports
godot-arch --report /tmp/godot-arch-results
```

**Notes:**

- The report will be saved as `godot-arch-report.json` in the specified directory
- If the directory doesn't exist, it will be created automatically
- The report includes detailed information about all test results, failures, and warnings
- Useful for CI/CD pipelines and automated analysis

**Report Format:**

```json
{
  "files_tested": 42,
  "files_failed": 3,
  "warnings": [...],
  "failed_reports": [...],
  "successful_reports": [...]
}
```

---

### `-h, --help`

**Description:** Display help information and exit.

**Example:**

```bash
godot-arch --help
godot-arch -h
```

---

### `-V, --version`

**Description:** Display version information and exit.

**Example:**

```bash
godot-arch --version
godot-arch -V
```

## Usage Examples

### Basic Usage

Lint the current directory with default configuration:

```bash
./godot-arch
```

### Custom Project Path

Lint a specific project directory:

```bash
./godot-arch --project ~/projects/my-godot-game
```

### Custom Configuration

Use a custom configuration file:

```bash
./godot-arch --config configs/strict-rules.yaml
```

### Generate Report

Run linting and save results to a report file:

```bash
./godot-arch --report ./build/reports
```

### Combined Options

Lint a specific project with custom config and generate a report:

```bash
./godot-arch \
  --project ~/projects/my-game \
  --config ~/configs/my-rules.yaml \
  --report ./ci-reports
```

### CI/CD Integration

Example GitHub Actions workflow:

```yaml
- name: Run GodotArch
  run: |
    chmod +x ./godot-arch
    ./godot-arch --report ./reports
```

## Exit Codes

- **0**: All checks passed successfully
- **1**: Tests executed with failed checks
- **2**: There was a runtime issue/error while executing

The exit code can be used in CI/CD pipelines to fail builds when linting issues are detected.
